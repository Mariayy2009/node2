В ходе выполнения практической работы я освоил(а) основные принципы работы с системой контроля версий Git. 
Я установил Git на свой компьютер, выполнил первоначальную настройку (указал имя и email), 
создал локальный репозиторий и научился добавлять файлы с помощью команд git add и git commit.

Также я создал удалённый репозиторий на GitHub, связал его с локальным репозиторием 
с помощью git remote add origin и загрузил изменения командой git push.

Я понял(а), что благодаря Git и удалённым репозиториям можно:
- Отслеживать все изменения в проекте
- Возвращаться к предыдущим версиям кода
- Создавать резервные копии проекта на GitHub
- Организовывать командную работу над проектом

Таким образом, Git является незаменимым инструментом для разработчика, обеспечивающим 
надёжное хранение кода и удобное управление версиями проекта.
